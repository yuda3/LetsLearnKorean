## 개선 방향

### 1. 다양한 오답 패턴 사용

#### ❌ 나쁜 예 (획일적)
```typescript
{
  question: '「写真を撮ってもらえますか？」は？',
  options: [
    '사진 찍어 주시겠어요?',  // 정답
    '사진 찍었어요',          // 과거형 (괜찮음)
    '사진 찍을래요',          // 의지 (괜찮음)
    '사진 좋아요',           // ❌ 패턴 반복! 의미 없음
  ],
}
```

#### ✅ 좋은 예 (다양하고 교육적)
```typescript
{
  question: '「写真を撮ってもらえますか？」は？',
  options: [
    '사진 찍어 주시겠어요?',     // 정답 (依頼)
    '사진 좀 찍어주세요',        // 유사 (カジュアル)
    '사진을 찍어 드릴까요?',     // 반대 (提供)
    '사진이 필요하시나요?',      // 다른 의미 (確認)
  ],
}
```

### 2. 카테고리별 특화된 오답 설계

#### Travel (여행)
- ✅ 교통수단 관련 혼동: 버스/기차/택시
- ✅ 방향 관련: 앞/뒤/왼쪽/오른쪽
- ✅ 숙박 관련: 체크인/체크아웃/예약
- ❌ 범용 패턴: ~좋아요, ~해요

예시:
```typescript
// "「タクシー乗り場はどこですか？」は？"
options: [
  '택시 타는 곳이 어디예요?',    // 正解
  '버스 타는 곳이 어디예요?',    // 교통수단 혼동 (教育的)
  '택시 내리는 곳이 어디예요?',  // 동작 혼동 (教育的)
  '택시 좋아요',                // ❌ 의미 없음
]
```

#### Restaurant (레스토랑)
- ✅ 음식 관련: 맵다/달다/짜다
- ✅ 주문 관련: 주문하다/계산하다/포장하다
- ✅ 양 관련: 많다/적다/더/덜

#### Shopping (쇼핑)
- ✅ 가격: 비싸다/싸다/할인
- ✅ 크기: 크다/작다/길다/짧다
- ✅ 색상: 빨간색/파란색/검은색

### 3. 학습 효과가 있는 오답 설계

#### 원칙
1. **유사 표현**: 학습자가 실제로 혼동할 수 있는 표현
2. **문법 차이**: 시제, 높임말, 부정형 등
3. **의미 차이**: 비슷하지만 뉘앙스가 다른 표현
4. **실수하기 쉬운 조사**: 이/가, 을/를, 에/에서

#### 예시 유형

**유형 1: 시제 혼동**
```typescript
options: [
  '예약했어요',      // 과거 (正解)
  '예약할게요',      // 未来
  '예약하고 있어요',  // 進行形
  '예약하세요',      // 命令形
]
```

**유형 2: 조사 혼동**
```typescript
options: [
  '공항까지 가주세요',  // まで (正解)
  '공항에 가주세요',    // に (목적지)
  '공항에서 가주세요',  // で (場所)
  '공항으로 가주세요',  // へ (方向)
]
```

**유형 3: 유사 단어**
```typescript
options: [
  '환불 받을 수 있어요?',  // 返金 (正解)
  '교환 받을 수 있어요?',  // 交換
  '교체 받을 수 있어요?',  // 交替
  '환승 받을 수 있어요?',  // 乗換
]
```

## 참고사항

### 좋은 오답의 조건
1. **교육적**: 학습자가 배울 점이 있음
2. **현실적**: 실제 사용 상황에서 혼동 가능
3. **다양성**: 같은 패턴 반복 최소화
4. **문맥적**: 질문의 맥락과 관련 있음

### 나쁜 오답의 예
- 전혀 관련 없는 단어
- 의미가 통하지 않는 조합
- 같은 어미만 바꾼 패턴 (특히 "~좋아요", "~해요")
- 문법적으로 말이 안 되는 표현
